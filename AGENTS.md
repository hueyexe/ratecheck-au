# AGENTS.md

## Repository Shape

- RateCheck public repo is the React/Vite SPA plus public generated artifacts under `public/`.
- Private data production lives in sibling/private repo `ratecheck-pipeline`; do not add crawler, LLM extraction, matching, or bank-specific scraping logic here.
- Frontend loads `public/rates.db` with sql.js; filtering/sorting should stay in parameterised SQL in `src/db.ts`, not JS array filtering.
- `public/rates.db` is latest snapshot only. Do not put historical snapshots back into the browser DB.
- `history.db` is local/private-pipeline state only. Never commit it or place it under `public/`.
- Analytics are precomputed into `public/analytics.json`; do not add heavy browser SQL/window-function analytics.
- Product rate history is exported as static JSON under `public/history/` and lazy-loaded by product pages.
- LLM/search exports are generated by the private pipeline: `public/llms.txt`, `public/llms-full.txt`, `public/*.md`, `public/robots.txt`, `public/sitemap.xml`.

## Commands

- Use Bun only for frontend work: `bun install`, `bun run dev`, `bun run lint`, `bun run build`, `bun test`.
- Run one frontend test with `bun test path/to/file.test.tsx`.
- Backend/data-pipeline commands run from the private `ratecheck-pipeline` repo, not this public frontend repo.
- Full frontend verification should include: `bun test`, `bun run lint`, `bun run build`, and `git diff --check`.

## Generated Data Flow

- Public generated artifacts are produced by the private `ratecheck-pipeline` repo and committed here by its GitHub Actions workflow.
- This repo deploys the generated static site; it does not run CDR aggregation or website verification.
- Update-rate commits intentionally trigger deploys; do not reintroduce `[skip ci]` there.
- `.github/workflows/deploy.yml` runs `bun run build` and deploys `dist/` to GitHub Pages.

## Routing And Static Assets

- `vite.config.ts` uses `base: "/"`; `src/main.tsx` uses `<BrowserRouter basename="/">`. Do not change to a subdirectory deployment.
- GitHub Pages SPA fallback uses `public/404.html` plus the restore script in `index.html`. The restore script must use `decodeURIComponent(p[1])` directly; prepending `window.location.pathname` causes doubled paths.
- Static fetches need `import.meta.env.BASE_URL` prefixes.

## Frontend Conventions

- Tailwind v4 is via `@tailwindcss/vite`; there is no `tailwind.config.js`.
- `verbatimModuleSyntax` is enabled, so use `import type` for type-only imports.
- DB row types intentionally use SQLite snake_case fields such as `RateRow.bank_name`.
- Use `accent-*` and `sand-*` tokens. Do not add `gray-*` or `indigo-*` classes.
- Use `.nums` for displayed rates, money, counts, and tabular numbers. Do not use `font-mono` for visual style.
- Active pills use `bg-accent-500 text-white rounded-full`; inactive pills use `border border-sand-200 rounded-full`.
- `MaterialIcon.tsx` is inline SVG only; do not add an icon package.
- `Dashboard.tsx` uses native SVG bars. Recharts is only for `AnalyticsPage.tsx`.
- Copy should be plain Australian English for non-technical homebuyers; avoid implying RateCheck covers every loan or gives financial advice.

## Site Feedback Worker

- Website feedback posts to `ratecheckau.homes/api/feedback`, handled by `worker/feedback-worker.ts`, then `site-feedback.yml` creates a public GitHub issue.
- Keep the GitHub dispatch token in Cloudflare Worker secret `GITHUB_DISPATCH_TOKEN`; never add it to frontend code or repo secrets exposed to the browser.

## Design Context

- Primary user: everyday Australian homebuyer; secondary users are brokers/refinancers who value dense filters, CSV, and analytics.
- Brand: clear, trustworthy, fresh; warm and direct, not bank-like or fintech-hype.
- Avoid gradient text/headers, glassmorphism, `border-t-4` accent cards, hover lift shadows, and jargon.
